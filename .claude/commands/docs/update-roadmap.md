docs/ROADMAP.md와 docs/TASKS.md의 태스크 완료 상태를 동기화하고 최신화해주세요.

## 수행 절차

1. **현재 코드 상태 파악**: git diff 및 git log를 확인하여 최근 완료된 작업을 파악합니다.

2. **ROADMAP.md 읽기**: `docs/ROADMAP.md` 파일을 읽어 현재 태스크 체크박스 상태를 확인합니다.

3. **TASKS.md 읽기**: `docs/TASKS.md` 파일을 읽어 현재 세부 태스크 상태를 확인합니다.

4. **완료 태스크 식별**: 아래 기준으로 태스크 완료 여부를 판단합니다.
   - 해당 태스크에서 명시한 파일이 존재하고 구현이 완료되었는지 확인
   - 테스트 태스크는 테스트 결과가 기록되어 있는지 확인
   - 빌드/린트 태스크는 실제 명령어를 실행하여 통과 여부 확인

5. **ROADMAP.md 업데이트**:
   - 완료된 태스크: `- [ ]` → `- [x]`로 변경
   - 서브 체크리스트도 함께 업데이트
   - ⚠️ 경고 마크가 있는 항목 중 해결된 것은 제거
   - Phase 상태 테이블의 상태 컬럼도 일치시키기 (완료 / 진행중 / 미시작)
   - Phase 제목에 상태 표기 일치시키기

6. **TASKS.md 업데이트**:
   - ROADMAP.md와 동일한 기준으로 완료된 태스크 체크 (`- [ ]` → `- [x]`)
   - TASKS.md의 세부 태스크가 ROADMAP.md의 상위 태스크와 정합성 유지
   - 현재 마일스톤이 완료되면 다음 마일스톤을 "현재 마일스톤"으로 승격
   - 완료된 마일스톤 섹션은 TASKS.md에서 제거 (ROADMAP.md에 이력 보존)

7. **정합성 검증**:
   - ROADMAP.md의 Phase 상태 테이블과 개별 태스크 체크박스가 일치하는지 확인
   - TASKS.md의 체크박스와 ROADMAP.md의 체크박스가 일치하는지 확인
   - 모든 태스크가 완료된 Phase는 상태를 "완료"로 변경
   - 하나라도 미완료 태스크가 있는 Phase는 "진행중"으로 유지

## 동기화 규칙

- ROADMAP.md = 전체 이력 (M1~M7 모든 태스크, PRD 추적 매트릭스 포함)
- TASKS.md = 세부 작업 목록 (M1~M7 전체, 완료 마일스톤은 요약, 진행중은 세부 태스크)
- 한쪽만 업데이트하지 않는다. 반드시 양쪽을 동시에 업데이트한다.
- 완료된 마일스톤의 섹션 제목에 "— 완료" 표기, 진행중은 "— 진행중", 미시작은 "— 미시작"

## 주의사항

- ROADMAP.md, TASKS.md 파일만 수정합니다. 코드 파일은 절대 수정하지 않습니다.
- 태스크 완료 여부가 불확실한 경우, 해당 태스크의 구현 파일을 직접 확인합니다.
- 변경 사항을 요약하여 사용자에게 보고합니다.
